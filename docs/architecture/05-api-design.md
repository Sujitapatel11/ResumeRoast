}

{
success: false,
error: {
code: "...",

}

3. Authentication
Protected endpoints require JWT Bearer tokens. Public endpoints include registration, login and
selected company information.
4. Validation Rules
Validate request bodies using Pydantic models. Reject malformed data with HTTP 422. Return
consistent error messages.
5. API Versioning
Initial release uses /api/v1/. Future breaking changes create /api/v2/ while maintaining backward
compatibility where feasible.
6. CTO Decisions
REST over GraphQL for V1. OpenAPI generated by FastAPI. Thin routers, service-layer business
logic, consistent response contracts
